Introduction

Bash, which stands for Bourne Again Shell, processes shell commands in plain text format. Bash Scripting is a way to write a program that the Bash shell will execute.

Bash Scripting is a vital component of the Linux Operating System. In Linux, we often interact with the computer using some sort of a command line terminal. We give the computer commands that creates files, directories, change into different directory locations, add users, change file permissions, etc... This is well and good but lots of these operations can be done without a human to type them manually. This is where "scripting", particularly Bash Scripting come in handy. Why Bash? Linux shell is often defaulted to use Bash, as oppose to other shell like sh, csh, and others. This documentation suggests a modular approach to learn how to start writing Bash scripts.

Bash scripting offers a way to automate tasks that make things easy for programmers, developers and administators. If you are curious on how many .sh (the Bash script file extension) your linux machine has, and is tempted to run the command: sudo find / -iname '*.sh' | wc -l #Do not run this unless you have a good computer and plenty of time! There are lots of .sh files in Linux and it signifies how essential for anyone working in Linux to know how to read and write Bash Scripts

What you should already know

In order to write effective Bash scripts, you have to be able to know how your script will behave when you run it manually. A bash script file contains commands that you should be able to run on the shell. It helps if you are already familiar with basic Linux commands, some idea of conditional statements, loops, file permissions, and other Linux concepts.

Linux Command Line Basics

Some Commands that you should be familiar with

  • To change directories: cd /bin cd someDirectoryinBin #This command takes you to /bin/someDirectoryinBin
  • To list contents of directories: ls . #This lists the contents of the current directory ls .. #This lists the contents of the parent directory ls /bin #This lists the contents of the directory /bin
  • To show environmental variables: env #This lists Linux environmental variables env | more #"|" takes the result of the 'env' command to the 'more' pager to have more control of viewing the results using pages. "|" is also called a pipe ls /bin #This lists the contents of the directory /bin
  • To create directories and files touch index.html #This creates an index.html file if it does not exist or updates the modification timestamp of the already existing index.html mkdir samplefolder #This creates a folder called 'samplefolder' in the current working directory
  • To add manage users: sudo adduser ethanhawke #This creates a user with a username of 'ethanhawke' sudo usermod -a -G ethanhawk actors#This adds the newly created user to the 'actors' group
  • There are lots of other basic Linux commands that will be shown in the example scipts that will be discussed in subsequent sections.

Variables

You use variables as symbolic names for values in your application. The names of variables conform to certain rules:

  • Variable names must start with an alphanumeric character or an underscore (_); subsequent characters can also be digits (0-9). Because Bash Script is case sensitive, letters include the characters "A" through "Z" (uppercase) and the characters "a" through "z" (lowercase).
  • Do not put spaces on either side of the equal sign when assigning value to variable. For example, the following is valid variable declaration
  • Variables names are case-sensitive, just like filenames.

Example Valid and Invalid Variable Names:

Valid: FIRST3LETTERS="ABC" FIRST_THREE_LETTERS="ABC" firstThreeLetters="ABC" Invalid: 3LETTERS="ABC" first-three-letters="ABC" first@Three@letters="ABC"

Declaring variables
You can declare a variable just like in this example: #!/bin/bash STR="Hello World!" echo $STR favorite_pastime="bath" echo "I am ${favorite_pastime}ing at the moment!" #The use of '{' and '}' with the variable name is sometimes needed if you need to add additional characters like 'ing' to the value of the variable

Line 2 creates a variable called STR and assigns the string "Hello World!" to it. Then the VALUE of this variable is retrieved by putting the '$' in at the beginning. Please notice (try it!) that if you don't use the '$' sign, the output of the program will be different, and probably not what you want it to be.

Another example is a variable that contains an array: my_array=("value 1" "value 2" "value 3") echo second element: ${my_array[1]} echo all elements: ${my_array[@]} echo number of elements: ${#my_array[@]}

Again, by preceding the variable name with '$', the value will be accessed.

Variables can hold the results of running Linux Commands

Here is an example:

!#/bin/bash #keeping the results of 'ls -al .bash*' to the variable FILES_TO_BACKUP FILES_TO_BACKUP=$(ls -al .bash*) #Make a backup of hidden bash files by copying the files we access from using a for loop to the values being held by the variable FILES_TO_BACKUP 
for f in $FILES_TO_BACKUP ;
do 
         cp $f backup/$f.bak 
done
Tests

Tests are vital parts of any conditional statements, like the 'if...else' statements:

File Operators - Tests

  • -d FILE #True if file is a directory
  • -e FILE #True if file exists
  • -f FILE #True if file exists and is a regular file
  • -r FILE #True if file is readable by you
  • -s FILE #True if file exists and is not empty
  • -w FILE #True if file is writeable by you
  • -x FILE #True if file is executable by you

String Operators - Tests

  • -z STRING #True if string is empty
  • -n STRING #True if string is not empty
  • STRING1 = STRING2 #True if strings are equal
  • STRING1 != STRING2 #True if strings are not equal

Arithmentic Operators - Tests

  • arg1 -eq arg2 #True if arg1 is equal to arg2
  • arg1 -ne arg2 #True if arg1 is not equal to arg2
  • arg1 -lt arg2 #True if arg1 is less than arg2
  • arg1 -le arg2 #True if arg1 is less than or equal to arg2
  • arg1 -gt arg2 #True if arg1 is greater than arg2
  • arg1 -ge arg2 #True if arg1 is greater than or equal arg2

if...else statement

The "if" statement:

In order to make decisions, we extend the use of the 'tests' that we just studied by providing conditional course of actions depending on the results of the tests.

if [ condition-is-true ] 
then
      command 1
      command 2
      command N 
fi

Example Bash Script utilizing the "if" statement:

#!/bin/bash 

if [[ -a ${file} ]]
then
  echo "True if file exists:"
fi
  
if [[ -r ${file} ]]
then
  echo "True if file exists and is readable:"
fi

Example Bash Script utilizing the "if-else" statement:

#!/bin/bash 

if [[ -a ${file} ]]
then
  echo "The file exists"
else
  echo "The file does not exist."
fi
  
if [[ -r ${file} ]]

  echo "The file exists and is readable:"

if [[ "$name" == "Ryan" ]] && ! [[ "$time" -lt 2000 ]]; then
  echo "Sleeping"
elif [[ "$day" == "New Year's Eve" ]] || [[ "$coffee_intake" -gt 9000 ]]; then
  echo "Maybe awake"
else
  echo "Probably sleeping"
fi
while statement
A while statement executes its statements as long as a specified condition evaluates to true. A while statement looks as follows: while (condition) statement If the condition becomes false, statement within the loop stops executing and control passes to the statement following the loop.

The condition test occurs before statement in the loop is executed. If the condition returns true, statement is executed and the condition is tested again. If the condition returns false, execution stops and control is passed to the statement following while.

The Syntax of a Bash While Loop is as follows:

while [ condition ]
do
   command1
   command2
   command3
done

An example 'While Loop' in Bash:

#!/bin/bash
  counter=$1
  factorial=1
  while [ $counter -gt 0 ]
  do
     factorial=$(( $factorial * $counter ))
     counter=$(( $counter - 1 ))
  done
  echo $factorial
Practical applications

FRST = File Renamer Sorter Tagger

      #!/bin/bash

#-------------------------
#       filename: fire_renamer_sorter_tagger.sh
#       description: This script renames files
#-------------------------

echo "This script by NostradmsX started."

file_extension_to_be_renamed=$1
some_description=$2

# creates a temporary file list of files that will be sorted, renamed, and tagged
ls -art *.$file_extension_to_be_renamed > temp_file

IFS=$'\n'       # make newlines the only separator

counter=1;

for i in $(cat < temp_file); do
    # Single-digit gets preceded by zero
    if [ "$counter" -le 9 ]; then
        mv $i 0${counter}_${some_description}.$file_extension_to_be_renamed
    else
        mv $i ${counter}_${some_description}.$file_extension_to_be_renamed
    fi

((counter=counter+1))
done
rm temp_file
echo "Script ended."
